Guava Collections使用体验

  • java

的确Java Collections Framework：Set,List,Map已经基本可以满足我们的要求；但当我们遇上一些比较复杂的业务需要时Guava Collections可以带来很酷的编码体验！！！
Guava Collections中的集合应该也是最成熟为人所熟知的一部分【不可变集合，新集合类型，强大的集合工具类，扩展集合工具类】。

JDK Collection

SET（无序，唯一）

HashSet()——为快速查找设计的Set（基本创建的都是HashSet）
TreeSet()——保存次序的Set, 底层为树结构。使用它可以从Set中提取有序的序列
LinkedHashSet()—–具有HashSet的查询速度，且内部使用链表维护元素的顺序(插入的次序)。于是在使用迭代器遍历Set时，结果会按元素插入的次序显示。

List(有序，不唯一)

ArrayList()—-由数组实现的List。允许对元素进行快速随机访问；插入和移除元素。比LinkedList开销要大很多。
LinkedList()—底层采用链表，插入和删除的效率高一点；随机访问则相对较慢。

Map(映射)

HashMap()

HashMap与HashTable可查看源码知实现功能与原理一致可互用；
区别：
1、HashMap继承AbstractMap类，实现Map接口；HashTable继承Dictionary，实现Map接口。
2、HashMap线程非安全；HashTable线程安全。
3、HashMap允许null值；HashTable不允许null值。

Guava Collection

Multiset ——文档计数，访问计数等

统计一个词在文档中出现了多少次，传统的做法是这样的：
这种写法很笨拙，也容易出错，并且不支持同时收集多种统计信息，如总词数。我们可以做的更好。

Guava提供了一个新集合类型 Multiset，它可以多次添加相等的元素。Multiset元素的顺序是无关紧要的：Multiset {a, a, b}和{a, b, a}是相等的；Multiset继承自JDK中的Collection接口，而不是Set接口，所以包含重复元素并没有违反原有的接口契约。

*[使用较多]Multimap ————一健多值集合映射

Guava的Multimap可以很容易地把一个键映射到多个值。换句话说，Multimap是把键映射到任意多个值的一般方式。可以用以下两种方式思考Multimap:
“键-单个值映射”的集合：
a -> 1 a -> 2 a ->4 b -> 3 c -> 5
或者”键-值集合映射”的映射：
a -> [1, 2, 4] b -> 3 c -> 5 这种方式是通过asMap()将视图返回为Map<K, Collection>形式；不会有任何空集合映射进去：一个键要么至少到一个值，要么根本就不在Multimap中。
*注：很少会直接使用Multimap接口，更多时候你会用ListMultimap或SetMultimap接口，它们分别把键映射到List或Set。

SetMultimap
SetMultimap<Object,Object> multMap = HashMultimap.create(); //对象创建

ListMultimap
ListMultimap<Object, Object> listMultMap = ArrayListMultimap.create();//对象创建

BiMap ———— 双向映射

传统上，实现键值对的双向映射需要维护两个单独的map，并保持它们间的同步。但这种方式很容易出错，而且对于值已经在map中的情况，会变得非常混乱。

BiMap<String, String> bimap = HashBiMap.create();//对象创建

可通过inverse()方法对集合进行健值反转BiMap<K, V>是特殊的Map：
● 可以用inverse()反转BiMap<K, V>的键值映射
● 保证值是唯一的，因此values()返回Set而不是普通的Collection
